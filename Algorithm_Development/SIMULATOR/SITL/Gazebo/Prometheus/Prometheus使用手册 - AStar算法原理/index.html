Prometheus使用手册 - AStar算法原理

A-star算法最初由Stanford研究院的Peter Hart、Nils Nilsson和Bertram Raphael于1968年提出，A-star算法是在Dijkstra算法的基础上引入了启发式函数（heuristic function），以优化搜索过程。A-star算法结合了Dijkstra算法的最短路径特性和启发式算法的目标导向特性,因其高效性和最优性而广泛应用于图搜索问题。

A-star算法的核心是将地图划分为一个个网格单元，即将地图分为由 𝑤 × ℎ个网格，因此最终寻找到的路径实际上是由一系列相邻的网格单元连接而成的。效果示意图如下:

注:上图中浅绿色为起点，深绿色为障碍物，橘红色为终点，紫色为探索过的区域，黄色为寻到到路径。

为便于理解，先定义以下变量:

  • OPEN List: 待检查的节点的集合，所有被发现但尚未被完全评估的节点都在这个优先级队列中。
  • CLOSE List: 所有已经被评估和处理过的节点的集合,这些节点已经被完全检查过，并且它们的最短路径成本已经被确定。
  • G(x): 从起点到当前节点 x 的实际成本
  • H(x): 从当前节点 x 到目标节点的启发式估计成本。
  • F(x): 表示从起点经过当前节点 x 到目标节点的总代价，其中包括从起点到 x 的实际成本 G(x) 和从 x 到目标节点的估计成本 H(x)，计算公式为 F(x) = G(x) + H(x)。

Step 1: 如下图所示，假设每个格子的水平和垂直距离都为1，则斜角距离为√2。从起点A开始，把它加入到一个OPEN List中，可以将OPEN List看成一个待检查的列表。

Step 2: 查看与当前节点相邻的8个方格 ，对于每一个相邻方格，如果它是可通行的（即不是障碍物），则将其加入到OPEN List中，并设置当前节点为该方格的父节点。将当前节点从OPEN List中移除，并将其加入到CLOSED List中,CLOSED List中的每个方格都无需再次考虑。

Step 3: 从OPEN List中选一个与父节点相邻的方格，这个方格是OPEN List中F(x)值最小的那个。其中H(x)使用曼哈顿方法计算距离。则父节点右边的F(x) = 1 + 33 = 34,而右上角和右下角的F(x) = √2 + 34; 因此右侧的值最小，将右侧点作为处理节点，移出OPEN List，移入CLOSED List。同理，以这个节点为中心，再对其周围八个节点进行探索，当周围的节点为障碍物或者已经在CLOSED List时，忽略这个点。若不为障碍物且不在CLOSED List，执行步骤4。

Step 4: 若当前处理节点的相邻格子已经在OPEN List中，则检查这条路径是否更优，即计算经由当前处理节点到达那个方格是否具有更小的G(x)值。如果没有，不做任何操作。相反，如果G(x)值更小，则把那个方格的父节点设为当前处理节点 ( 我们选中的方格 ) ，然后重新计算那个方格的 F(x) 值和 G(x) 值。 若当前处理节点的相邻格子不在OPEN List中，那么把它加入，并将它的父节点设置为该节点。

Step 5: 循环步骤2 步骤3 步骤4，直到把终点也加入到了OPEN List中，当终点从OPEN List中拿到CLOSED List时，认为找到障碍物。

Step 6: 从CLOSED List中回溯起点到终点的路径。
